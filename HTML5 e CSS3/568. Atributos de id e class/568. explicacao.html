Atributos de id e class

Por que digitar (e não copiar) o código

Muita gente reclama de não deixar o código de algumas sessões para copiar. O motivo é simples: não queremos que você copie e cole no editor sem fazer nada. Você precisa trabalhar no curso — digitar tudo o que está sendo mostrado, testar e sentir como é fazer o código. Por isso, digite tudo no seu computador.

Anatomia da tag

Uma tag tem abertura, conteúdo dentro e fechamento. Tudo o que você colocar dentro da tag fica “dentro” dela. Lembre-se: você não pode fechar a tag “por fora” do conteúdo — por exemplo, não pode fechar o ‹h1› antes de terminar o texto que é filho dele, senão o navegador não entende direito e o layout quebra.

Se existe hierarquia (ex.: ‹body› tem filhos como ‹h1›), você não pode fechar o pai antes de fechar o filho. Fechar o ‹body› antes do ‹h1› está incorreto. É muito importante sempre fechar na ordem correta. Algumas tags não têm corpo; outras têm — as que têm corpo precisam ser fechadas.

Atributos em HTML

Em HTML podemos ter atributos: alguns com valor (recomendável colocar entre aspas) e alguns sem valor (o navegador trata como booleano true). Os mais famosos são o id e a class, que ficam na abertura da tag.

Atributo id

O id é um identificador único daquele elemento. Você não pode ter duas tags na mesma página com o mesmo id. O valor deve ser único (ex.: cabecalho-um, cabecalho-dois). Quando for colocar o valor, use aspas (recomendável).

  • Um único id por página para cada valor.
  • Exemplo: ‹h1 id="cabecalho-um"› e ‹h1 id="cabecalho-dois"›.

Padrão de nomenclatura (id e class)

Um padrão muito comum na internet é usar traço entre palavras e sem acento: cabecalho-um, cabecalho-dois, fundo-vermelho. Você pode usar como preferir (ex.: cabecalho1), mas esse padrão com traço é o que costumamos seguir. Se for uma palavra só, pode ser só a palavra (ex.: cabecalho).

Atributo class

A class também é um identificador de elementos, mas diferente do id: as classes podem se repetir. Você pode ter vários elementos com a mesma classe e um mesmo elemento pode ter mais de uma classe (separadas por espaço).

  • Exemplo: class="fundo-vermelho" em vários ‹h1›.
  • Várias classes no mesmo elemento: class="fundo-vermelho borda-esquerda".
  • Resumo: id = único na página; class = pode repetir e um elemento pode ter várias.

Tag ‹style› e CSS

A tag ‹style› geralmente fica dentro do ‹head›. Dentro dela colocamos o CSS. Precisamos de CSS para “fazer algo” com o HTML; senão fica só estrutura. Aqui vemos um básico de CSS; em outra seção dá para aprofundar.

Seletores CSS

Para aplicar estilo a um elemento, precisamos selecioná-lo. No CSS:

  • Por nome da tag: h1 { } — seleciona todas as tags ‹h1›. O que você colocar dentro das chaves vale para todos.
  • Por id: #cabecalho-dois { } — seleciona o elemento cujo id é cabecalho-dois. O # é o nome do id. Ex.: #cabecalho-dois { background: purple; }.
  • Por class: .fundo-vermelho { } — seleciona todos os elementos que têm a classe fundo-vermelho. O . (ponto) é o nome da classe. Ex.: .fundo-vermelho { background: brown; }. Todos com essa classe recebem o estilo.

Exemplos práticos: selecionar h1 e dar background: red; — todos os h1 ficam com fundo vermelho. Selecionar p e dar background: blue; — todos os parágrafos ficam com fundo azul. Isso não afeta outras tags. Para um elemento específico, use o id; para vários, use class.

Especificidade

O CSS tem especificidade: a “força” do seletor. Se duas regras tiverem a mesma especificidade (ex.: duas com seletor h1), o que prevalece é o que o navegador lê por último. O seletor por id tem mais força que o por class. Por isso, se você colocar estilo em um elemento por id, fica mais difícil sobrescrever depois só com classe — a não ser que use recursos mais avançados (que não são recomendáveis forçar com !important em tudo).

Recomendação: Prefira classes quando quiser selecionar vários elementos (ex.: todos os títulos com fundo marrom). Use id só quando precisar de um elemento muito específico (ex.: um único cabeçalho com fundo roxo). Assim você evita conflitos de especificidade e mantém o código mais fácil de manter.

Resumo rápido

  • Digite o código, não só copie — para praticar e fixar.
  • Anatomia da tag: abrir → conteúdo (e filhos) → fechar na ordem correta.
  • id: identificador único na página; no CSS use #nome-do-id.
  • class: pode repetir e um elemento pode ter várias; no CSS use .nome-da-classe.
  • Padrão comum: traço entre palavras, sem acento (ex.: cabecalho-um).
  • Seletores: por tag (h1), por id (#id), por class (.class).
  • Especificidade: mesmo nível = último vence; id tem mais força que class.
  • Preferir classes para vários elementos; id para elemento único quando necessário.
