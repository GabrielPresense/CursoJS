Estrutura do HTML e Meta ViewPort

Atalho no editor (VS Code)

Você não precisa criar toda a estrutura manualmente toda vez. Se estiver usando o VS Code com a extensão que foi sugerida, basta digitar uma exclamação ! e apertar Enter — o editor já gera o documento HTML com a mesma estrutura que vimos anteriormente.

O Emmet pode colocar o título em inglês (Document). Vale corrigir para lang="pt-BR" no ‹html› e colocar um título como "Meu HTML" (ou o que você quiser) na tag ‹title›.

HTML é case insensitive

Uma coisa importante: o HTML é case insensitive. Não faz diferença você usar letra maiúscula ou minúscula nas tags. Tanto faz.

Tags com corpo: h1 e p

As tags ‹h1› e ‹p› andam quase que juntas: h1 é um título, p significa parágrafo. Sempre que estamos escrevendo HTML precisamos de algum "place holder" para ver algo na página.

  • Tag com corpo → você abre a tag, coloca o conteúdo (e eventualmente outras tags dentro), e fecha a tag. Ex.: ‹p› texto ‹/p›.
  • Se você quebrar a linha no código, o texto continua dentro do parágrafo no navegador. O que importa é que a tag com corpo seja fechada.
  • Regra: abrir tag → colocar o conteúdo (e filhos) → fechar a tag. Não pode fechar o ‹body› antes de fechar o ‹p› ou o ‹h1›, senão o código fica incorreto.

Tags sem corpo (ex.: br)

Existem tags que não têm corpo. Exemplo: ‹br› (quebra de linha). Visualmente não tem "estilo" grande no corpo; ela é inline — o efeito dela é só aplicado no texto, não atrapalha outras coisas ao lado.

Como a tag ‹br› não tem corpo, você não precisa fechar ela. Vários ‹br› seguidos significam várias quebras de linha. Ex.: "Luiz Otávio" ‹br› "em Salvador" — cada parte fica em uma linha. Se tirar o ‹br›, o texto fica na mesma linha.

Meta viewport — para quê serve?

Quando o Emmet gera o documento, ele coloca uma tag a mais: a meta viewport. Não existe na documentação oficial do HTML; foi introduzida pela Apple no Safari. Não faz diferença para a "história" da especificação, mas todos os navegadores conhecidos aceitam. Ela é mais avançada do que a gente precisa neste momento, mas é importante saber para quê serve.

  • Problema em celular: os navegadores para celular usam uma janela virtual chamada viewport para renderizar a página. Essa janela pode ser maior que a tela do dispositivo (por exemplo, mais larga).
  • Resultado: sem a meta viewport, você tem que arrastar a página para o lado para ver o conteúdo — a página foi renderizada em um viewport maior que a tela.
  • Solução: a Apple introduziu a meta tag viewport. Com content="width=device-width" (e initial-scale=1.0), estamos dizendo que a largura do viewport deve ser a mesma da tela do dispositivo. Assim, a janela virtual se encaixa no tamanho da tela e não precisamos ficar arrastando.

Resumindo: essa tag está aí para dispositivos móveis. Com ela, o site se ajusta ao tamanho da tela do telefone.

Dica: No Chrome você pode simular um dispositivo móvel (F12 → ícone de celular / "Toggle device toolbar"). Se tirar a meta viewport e atualizar, a página pode ser renderizada de um jeito que obriga a arrastar. Colocando a tag de volta e atualizando, ela se ajusta à largura da tela.

Resumo rápido

  • Use ! no VS Code para gerar a estrutura do HTML.
  • Corrija o idioma para lang="pt-BR" e o título em ‹title› (ex.: "Meu HTML").
  • HTML é case insensitive.
  • Tags com corpo (h1, p): abrir → conteúdo → fechar. Tags sem corpo (br): não precisam fechar.
  • Meta viewport: faz a página se ajustar à largura da tela em dispositivos móveis.
